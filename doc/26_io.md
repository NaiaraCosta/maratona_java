# 📘 Guia — Java IO: `File`, `FileWriter`, `FileReader`, `BufferedWriter`, `BufferedReader` e Diretórios

Introdução prática às **classes utilitárias de IO do Java**, focando em criação, escrita, leitura e manipulação de arquivos e diretórios.

---

## 🧠 Resumo Contextualizado 🤯📌

💡 **Java IO (`java.io`)** fornece classes **altamente coesas**, cada uma com um papel específico no ecossistema de entrada e saída.  
Neste guia, são exploradas as classes:

- `File` → representação de arquivos e diretórios
- `FileWriter` / `FileReader` → escrita e leitura **low level**
- `BufferedWriter` / `BufferedReader` → escrita e leitura **otimizadas com buffer em memória**

O objetivo é entender **como os arquivos realmente funcionam**, visualizando resultados de forma prática e concreta.

---

## 🧩 Blocos Semânticos 🧠🔎

---

### 📁 Classe `File` — Fundamentos

A classe `File` **não cria arquivos automaticamente**, ela apenas representa um caminho no sistema.

```java
    File file = new File("file.txt");
```

✅ Para criar o arquivo de fato:

- Use `createNewFile()`
- Retorna um `boolean`
- Lança exceção `checked`

```java
    file.createNewFile();
```

🧠 **Comportamento importante:**

- Se o arquivo já existir → retorna `false`
- Não sobrescreve conteúdo existente

---

### ❌ Exclusão e Verificação

Principais métodos:

- delete() → remove arquivo
- exists() → verifica existência

✅ **Boa prática:** sempre verificar antes de deletar

Exemplo:
```java
    if (file.exists()) {
        file.delete();
    }
```

---

### 📍 Caminhos e Metadados

- `getPath()` → nome ou caminho relativo
- `getAbsolutePath()` → caminho completo
- `isFile()` /` isDirectory()`
- `isHidden()`
- `canRead()` / `canWrite()`

🕒 Última modificação:

Exemplo:
```java
    long millis = file.lastModified();
```

Conversão moderna com `java.time`:

Exemplo:
```java
    Instant.ofEpochMilli(millis).atZone(ZoneId.systemDefault()).toLocalDateTime();
```

---

### ✍️ Escrita com FileWriter

Uso **direto** e **low level** para **escrita**.

Exemplo:
```java
    try (FileWriter fw = new FileWriter(file)) {
        fw.write("Conteúdo do arquivo");
        fw.flush();
    }
```

⚠️ Pontos críticos:

- **Sempre fechar o recurso**
- Usar `try-with-resources`
- `flush()` garante que o buffer seja persistido

#### 🧩 Modo append:

Exemplo:
```java
    new FileWriter(file, true);
```

---

### 📖 Leitura com `FileReader`

Leitura **caractere por caractere**

Exemplo:
```java
    int i;
    while ((i = fr.read()) != -1) {
        System.out.print((char) i);
    }
```

📌 Observações:

- `read()` retorna int
- **-1** indica **fim do arquivo**
- Conversão necessária para char

---

### 🚀 Escrita Otimizada com `BufferedWriter`

`BufferedWriter` encapsula um `Writer`, geralmente um `FileWriter`.

Exemplo:
```java
    try (BufferedWriter bw = new BufferedWriter(new FileWriter(file))) {
        bw.write("Linha de texto");
        bw.newLine();
        bw.flush();
    }
```

✅ Vantagens:

- Escrita em **memória**
- Menos acesso ao disco
- Melhor performance em arquivos grandes

🧠 `newLine()` usa o `line separator` **do sistema operacional**.

---

### 📚 Leitura Otimizada com BufferedReader

Leitura **linha por linha**:

Exemplo:
```java
    String linha;
    while ((linha = br.readLine()) != null) {
        System.out.println(linha);
    }
```
📌 Diferença-chave:

- `readLine()` retorna `null` no fim do arquivo
- Muito mais performático que leitura por caractere

---

### 📂 Trabalhando com Diretórios (mkdir e renameTo)

Criar diretório:

Exemplo:
```java
  JavaFile dir = new File("pasta");
  dir.mkdir();
```

Criar arquivo dentro do diretório:

Exemplo:
```java
    File arquivo = new File(dir, "arquivo.txt");
    arquivo.createNewFile();
```

Renomear arquivo ou diretório:

Exemplo:
```java
  File novo = new File(dir, "novo_nome.txt");
  arquivo.renameTo(novo);
```  
  
⚠️ Importante:

- `renameTo()` depende do caminho de destino
- Pode mover o arquivo se o diretório mudar
- Retorna `boolean`

---

## 🚀 Síntese Final ✅🧠

- `File` representa caminhos, não cria automaticamente
- `createNewFile()` e `mkdir()` retornam `boolean`
- `FileWriter` e `FileReader` são `low level`
- `BufferedWriter` e `BufferedReader` usam `buffer` **em memória**
- `flush()` garante persistência dos dados
- `read() retorna int, readLine() retorna `String`
- Diretórios e arquivos usam a mesma classe: `File`
- `renameTo() pode mover e renomear
- `try-with-resources` é obrigatório para `IO`
