## 🧩 Blocos Semânticos 🧠🔎

---

### 🔒 Encapsulamento e Acoplamento

📌 **Encapsulamento** significa **esconder os detalhes internos** de uma classe e permitir acesso **somente por meio de regras controladas**.
📌 **Acoplamento** representa o quanto uma classe **depende da implementação interna da outra**.

✅ **Alta coesão** → cada classe possui um único objetivo claro
✅ **Baixo acoplamento** → mudanças internas **não quebram** código externo

Exemplo de classe sem encapsulamento:

```java
    public String nome;
    public int idade;
```

❌ Problemas desse modelo:

- Qualquer classe pode alterar valores para estados inválidos (-1, `null`, etc.)
- Não há controle, validação ou garantia de consistência
- A classe fica frágil e exposta

---

### 🔐 Modificador private

Ao usar `private`, os atributos **não podem ser acessados diretamente fora da classe**

Exemplo:

```java
    private String nome;
    private int idade;
```

✅ Benefícios diretos:

- Protege o estado interno
- Reduz acoplamento
- Centraliza validações
- Permite alterar a lógica interna sem afetar quem usa a classe

📌 O atributo deixa de ser um detalhe público e passa a ser **uma implementação interna**

---

### 🧩 Métodos set — Escrita Controlada

Convenção dos métodos set:

- Nome inicia com `set`
- Recebe um parâmetro
- Retorno `void`

Exemplo:

```java
    public void setIdade(int idade) {
        if (idade < 0) {
            System.out.println("Idade inválida");
            return;
        }
        this.idade = idade;
    }
```

📌 Pontos importantes:

- `this.idade` se refere ao **atributo da classe**
- `idade` (sem `this`) é o parâmetro local
- Permite validações sem quebrar quem chama o método

✅ **Encapsulamento** mantém o **controle dentro da classe**

---

### 🧱 Encapsulamento na Prática

Código externo interagindo corretamente

Exemplo:

```java
    Pessoa p = new Pessoa();
    p.setNome("Jiraya");
    p.setIdade(70);
    System.out.println(p.getNome());
```

✅ Não acessa atributos diretamente
✅ Não quebra se a lógica interna mudar
✅ Mantém um **contrato estável** entre classes

📌 Quem usa a classe **não precisa saber como ela funciona internamente**

---

### 🚫 Nem Todo Atributo Precisa de `set`

⚠️ Alguns atributos:

- Devem ser apenas de leitura
- São calculados internamente
- Não devem ser alterados externamente

Exemplo:

```java
    private double mediaSalarial;
    public double getMediaSalarial() {
        return mediaSalarial;
    }
// ❌ Não existe setMediaSalarial()
```

✅ Evita corrupção de estado
✅ Garante consistência dos dados
✅ Impõe regras claras de uso

---

### 🔁 Sobrecarga de Métodos

📌 Sobrecarga ocorre quando métodos possuem:

- O **mesmo nome**, mas quantidade de parâmetros diferente ou tipos de parâmetros diferentes

Exemplo:

```java
    public void init(String nome, String tipo, int episodios) { }
    public void init(String nome, String tipo, int episodios, String genero) { }
```

✅ A sobrecarga:

- **Não depende do tipo de retorno**
- **Não depende de modificadores** (`public`, `private`)
- **Depende exclusivamente da assinatura** (nome + parâmetros)

📌 Permite evoluir a API sem quebrar código existente

---

### 🏗️ Construtores

📌 Um **construtor**:

- Tem o **mesmo nome da classe**
- **Não possui tipo de retorno**
- É executado automaticamente na criação do objeto

Exemplo:

```java
    public Anime(String nome) {
        this.nome = nome;
    }
```

⚠️ Regras importantes:

- Se **nenhum construtor é criado**, o Java gera um **padrão vazio**
- Se **qualquer construtor** for criado, o **padrão não é mais gerado**

📌 Construtores são o **ponto oficial de inicialização do objeto**

#### 🔁 Sobrecarga de Construtores

Exemplo:

```java
    public Anime(String nome) {
        this.nome = nome;
    }
```

✅ Permite criar objetos de maneiras diferentes.
✅ Apoia flexibilidade e evoluções futuras.
✅ Evita duplicação de lógica.

Delegação entre construtores com `this(...)`:

```java
    public Anime(String nome, String tipo, int ep) {
        this(nome, tipo);
        this.episodios = ep;
    }
```

📌 Regras do `this(...):`

- Pode ser usado **apenas em construtores**
- Deve ser a **primeira linha**

---

### 🧱 Blocos de Inicialização  (Instância)

São **executados sempre que um objeto é criado**, antes do construtor:

Exemplo:

```java
    System.out.println("Bloco de inicialização");
```

✅ Ideal para l**ógica comum a todos os construtores**
✅ Evita duplicação de código

📌 Executa antes do corpo do construtor

---

### 🧬 Modificador `static` — Conceito-Chave

📌 O modificador `static` faz atributos e métodos pertencerem à classe, não ao objeto

Exemplo:

```java
    public static double VELOCIDADE_LIMITE = 250;
```

✅ Compartilhado por todas as instâncias
✅ Existe independentemente de objetos

Uso:

```java
    Carro.VELOCIDADE_LIMITE = 180;
```

📌 Não é necessário criar um objeto para acessar membros `static`

---

### ⚠️ static vs Instância

| Contexto          | Pode acessar instância? |
|-------------------|-------------------------|
| Método não static | ✅ Sim                   |
| Método static     | ❌ Não                   |

📌 Métodos static:

- Não possuem `this`
- Não conhecem atributos de instância
- Existem mesmo sem objetos criados

---

### ⚙️ Métodos static

Indicados quando:

- O comportamento **não depende de estado do objeto**
- São funções utilitárias
- Representam regras da classe como um todo

Exemplo:
```java
    public static void main(String[] args) { }
```

✅ Executado sem instância
✅ Ponto de entrada da aplicação Java

---

### 🧯 Ordem de Inicialização

📌 Ordem real de execução em Java:

1. Blocos static
2. Atributos static
3. Blocos de inicialização (instância)
4. Construtor
5. Métodos

📌 Entender essa ordem evita bugs difíceis de rastrear

---

## 🚀 Síntese Final ✅🧠

- **Encapsulamento** protege o estado interno.
- `private` reduz acoplamento.
- `get/set` são contratos estáveis.
- Nem todo atributo deve ter `set`.
- Sobrecarga evita quebras de código.
- Construtores inicializam o objeto.
- `this(...)` delega entre construtores.
- Blocos de inicialização evitam duplicação.
- `static` pertence à classe, não à instância.
- Métodos `static` não acessam instância.
- **Código limpo** é **manutenível**, **seguro** e **escalável**.
