## 🧩 Blocos Semânticos 🧠🔎

---

### 🔒 Encapsulamento e Acoplamento

📌 **Encapsulamento** = esconder dados internos e permitir acesso controlado.  
📌 **Acoplamento** = o quanto uma classe depende dos detalhes da outra.

✅ **Alta coesão** → cada classe faz poucas coisas bem definidas  
✅ **Baixo acoplamento** → mudanças internas **não quebram** código externo

Exemplo:
```java
    public String nome;
    public int idade;
```

❌ Qualquer classe pode alterar valores inválidos (-1, `null`, etc.)

---

### 🔐 Modificador private

Ao usar `private`, os atributos **não podem ser acessados diretamente** fora da classe:

Exemplo:
```java
    private String nome;
    private int idade;
```

✅ Protege o estado interno
✅ Reduz acoplamento
✅ Permite validações internas

---

### 🧩 Métodos set — Escrita Controlada

Convenção:

- Nome inicia com `set`
- Recebe parâmetro
- Retorno void

Exemplo:
```java
    public void setIdade(int idade) {
        if (idade < 0) {
            System.out.println("Idade inválida");
            return;
        }
        this.idade = idade;
    }
```

📌 `this` diferencia atributo da variável local
📌 Permite validações sem quebrar quem chama

---

### 🧱 Encapsulamento na Prática

📌 Código externo:

Exemplo:
```java
    Pessoa p = new Pessoa();
    p.setNome("Jiraya");
    p.setIdade(70);
    System.out.println(p.getNome());
```

✅ Não acessa atributos diretamente
✅ Não quebra se lógica interna mudar
✅ Mantém **contrato estável**

---

### 🚫 Nem Todo Atributo Precisa de set

⚠️ Alguns atributos:

- Só leitura
- Uso interno
- Derivados de cálculo

Exemplo:
```java
    private double mediaSalarial;
    
    public double getMediaSalarial() {
        return mediaSalarial;
    }
    // ❌ Sem setMediaSalarial
```

✅ Evita corrupção de estado
✅ Garante consistência

---

### 🔁 Sobrecarga de Métodos

📌 **Mesmo nome**, mas:

* Quantidade de parâmetros diferente **ou**
* Tipos diferentes

Exemplo:
```java
    public void init(String nome, String tipo, int episodios) { }

    public void init(String nome, String tipo, int episodios, String genero) { }
```

✅ Não depende de retorno
✅ Não depende de `public` ou `private`
✅ Baseado **apenas na assinatura**

---

### 🏗️ Construtores

📌 Construtor:

- **Mesmo nome da classe**
- **Sem tipo de retorno**
- Executado na criação do objeto

Exemplo:
```java
    public Anime(String nome) {
        this.nome = nome;
    }
```

⚠️ Se **nenhum construto**r é criado, o Java gera um padrão automaticamente
⚠️ Se você cria um, o padrão não é mais gerado

---

### 🔁 Sobrecarga de Construtores

Exemplo:
```java
    public Anime(String nome) {
        this.nome = nome;
    }
```

✅ Permite diferentes formas de criação
✅ Evita duplicação de código
💡 Delegação com this(...):

Exemplo:
```java
    public Anime(String nome, String tipo, int ep) {
        this(nome, tipo);
        this.episodios = ep;
    }
```

📌 `this`(...):

- Apenas em **construtores**
- Deve ser a **primeira linha**

---

### 🧱 Blocos de Inicialização (Instância)

Executados **sempre que um objeto é criado, antes do construtor:**

Exemplo:
```java
    {
    System.out.println("Bloco de inicialização");
    }
```

✅ Ideal para lógica comum a **todos os construtores**
✅ Evita duplicação

---

### 🧬 Modificador static — Conceito-Chave

📌 `static` faz atributos/métodos **pertencerem à classe**, não ao objeto.

Exemplo:
```java
    public static double VELOCIDADE_LIMITE = 250;
```

✅ Compartilhado por todas as instâncias
✅ Pode ser acessado sem objeto:

Exemplo:
```java
    Carro.VELOCIDADE_LIMITE = 180;
```

---

### ⚠️ static vs Instância

| Contexto            | Pode acessar instância? |
|---------------------|-------------------------|
| Método não estático | ✅ Sim                  |
| Método `static`     | ❌ Não                  |

📌 `static` **não tem** `this`
📌 Executa mesmo sem objeto existir

---

### ⚙️ Métodos static

Indicados quando:

- Não dependem de atributos da instância
- São utilitários
- Representam comportamento da classe

Exemplo:
```java
    public static void main(String[] args) { }
```

✅ Executado sem instância
✅ Ponto de entrada do programa

---

### 🧯 Ordem de Inicialização 

1. Blocos static
2. Atributos static
3. Blocos de inicialização (instância)
4. Construtor
5. Métodos

---

### 🚀 Síntese Final ✅🧠

- **Encapsulamento protege estado interno**
- `private` reduz acoplamento
- `get/set` são contratos estáveis
- Nem todo atributo deve ter `set`
- Sobrecarga evita quebras de código
- Construtores inicializam o objeto
- `this(...)` delega entre construtores
- Blocos de inicialização evitam duplicação
- `static` pertence à classe, não à instância
- Métodos `static` não acessam instância
- Código limpo = **manutenível + escalável**
